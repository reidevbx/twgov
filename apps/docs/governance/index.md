# 治理

## 治理模型

gov.tw Design System 採用開放治理模型，鼓勵跨機關協作與社群參與。

## 變更流程

### 1. 提案

任何人都可以透過 GitHub Issue 提出新元件、模式或修改建議。

### 2. 討論

提案會在 Issue 中公開討論，收集各方意見與使用情境。

### 3. RFC（Request for Comments）

重大變更需經過 RFC 流程：

1. 撰寫 RFC 文件，說明問題、方案與影響
2. 公開審查期（至少兩週）
3. 核心團隊審議與決議

### 4. 實作與審核

- 實作需符合設計系統的所有標準
- 包含完整的文件與測試
- 通過無障礙檢查

### 5. 發布

合併後會納入下一個版本發布。

## 版本管理

本設計系統遵循 [Semantic Versioning](https://semver.org/)：

- **主版本**（Major）：不相容的 API 變更
- **次版本**（Minor）：新增向下相容的功能
- **修訂版**（Patch）：向下相容的問題修正

## 參與方式

- **回報問題**：透過 GitHub Issues
- **提出建議**：透過 GitHub Discussions
- **提交程式碼**：透過 Pull Request
