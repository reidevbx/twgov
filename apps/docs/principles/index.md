# 設計原則

本設計系統以六項核心原則作為所有設計與技術決策的依據。這些原則參照 [GOV.UK Design Principles](https://www.gov.uk/guidance/government-design-principles)、[WCAG 2.2](https://www.w3.org/TR/WCAG22/) 及台灣公共數位服務的實務經驗制定，適用於所有採用本系統的機關與團隊。

1. [以使用者為中心](/principles/people-first) — 服務設計以民眾需求為起點，而非機關內部流程
2. [無障礙與包容性](/principles/everyone-counts) — 公共服務必須對所有使用者可用，無障礙是基本要求
3. [降低使用者負擔](/principles/simplicity-is-designed) — 系統端承擔複雜度，使用者端獲得簡潔體驗
4. [一致性與可預期性](/principles/trust-through-consistency) — 跨服務的一致互動模式建立使用者信任
5. [資料驅動的持續改善](/principles/keep-improving) — 以量化指標與使用者回饋持續優化服務品質
6. [開放與協作](/principles/work-in-the-open) — 程式碼、設計決策與研究成果公開共享
